## Why

投资相关事项（财报披露、解禁、分红、会议、个人跟踪节点等）需要按日期管理，并且能够在当天快速回顾/编辑。当前缺少一个按月查看、按日维护的“万年历”入口，导致记录分散、不可追溯、也无法形成账号级沉淀。

## What Changes

- 新增一个“投资日历”界面：按月展示的万年历视图，支持跨月/跨年浏览。
- 点击某天弹出编辑窗口：
  - 新增/编辑/删除日历项（内容、提醒消息 group、提醒消息内容）。
  - 在弹窗内展示该日期下已添加的日历项列表，点击列表项可加载到表单进行编辑、保存或删除。
- 所有日历项按账号持久化保存到“账号数据库”（account DB），确保不同账号的数据隔离与可迁移。

## Capabilities

### New Capabilities
- `investment-calendar-ui`: 投资日历月视图与日期弹窗交互（查看、选择日期、列表展示、表单编辑）。
- `investment-calendar-crud`: 投资日历项的增删改查能力（面向 UI 的 API/服务层接口与校验）。
- `investment-calendar-account-db-storage`: 投资日历项在账号数据库中的存储与迁移（含索引/约束，支持 MySQL 与 SQLite 语义一致）。

### Modified Capabilities
- （空）

## Impact

- **Web UI**：新增页面路由/菜单入口；新增月历组件与弹窗表单；需要通用的列表/表单校验与通知提示。
- **Backend/API**：新增日历项 CRUD 接口（按账号隔离）；新增提醒 group 与提醒内容的字段校验与长度限制。
- **Database**：账号库新增日历相关表/索引；必须同步 MySQL 与 SQLite 的建表/迁移逻辑以保持一致性。
