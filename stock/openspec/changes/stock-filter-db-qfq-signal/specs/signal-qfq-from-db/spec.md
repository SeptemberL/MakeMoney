## ADDED Requirements

### Requirement: 信号计算必须使用数据库行情并转换为前复权口径

系统 MUST 在进行任何信号计算时，从数据库读取该股票的行情数据（未复权口径）；并同时从数据库 `adj_factor` 表读取与这些行情记录交易日对齐的前复权因子，将价格字段转换为前复权口径后再计算指标与信号。

#### Scenario: 使用 DB 行情 + DB 前复权因子完成计算
- **WHEN** 系统为某股票、某交易日区间执行信号计算
- **THEN** 系统 MUST 从数据库读取该区间的未复权行情与对应交易日的前复权因子，并在转换后使用前复权价格序列进行计算

### Requirement: 复权因子与行情必须按交易日对齐

系统 MUST 以 `trade_date`（或等价字段）作为主键对齐未复权行情与前复权因子；对齐后才允许用于计算。

#### Scenario: 交易日严格对齐
- **WHEN** 行情记录的 `trade_date` 与复权因子记录的 `trade_date` 存在不一致或无法一一匹配
- **THEN** 系统 MUST 将该区间标记为对齐失败并按缺失策略处理（跳过或回退），且不得静默当作已复权数据继续计算

### Requirement: 缺失复权因子时必须显式降级或跳过

当复权因子缺失导致无法完成前复权转换时，系统 MUST 采用明确且可配置的策略处理，并在输出中暴露该策略与覆盖情况。

#### Scenario: 默认策略为跳过并标记不可计算
- **WHEN** 某股票在目标区间内存在任意交易日缺失前复权因子
- **THEN** 系统 MUST 将该股票的该次信号计算结果标记为 `insufficient_data`（或等价状态）并跳过信号触发

#### Scenario: 可选策略回退到未复权计算但必须标注
- **WHEN** 配置启用“raw_fallback”，且目标区间存在缺失前复权因子
- **THEN** 系统 MAY 使用未复权价格序列完成计算，但 MUST 在输出中标注 `adjustment=raw_fallback` 且暴露缺失覆盖率

### Requirement: 信号结果必须暴露数据口径元数据

系统 MUST 在信号计算输出（或可观测日志）中记录本次计算的数据来源与复权口径信息，以便回溯与排错。

#### Scenario: 输出包含口径字段
- **WHEN** 系统完成一次信号计算（无论是否触发）
- **THEN** 输出中 MUST 包含 `data_source=db`，以及 `adjustment=qfq|raw|raw_fallback`（或等价字段），并包含复权因子覆盖情况
