## Why

量化页（`/quant`）已预留「策略回测」卡片，但尚无可用功能；需要在本地股票分析栈内提供可迭代的回测能力，并在量化入口暴露，便于验证策略与后续扩展因子/组合模块。

## What Changes

- 新增**量化回测子系统**：基于项目已有行情与指标能力，提供策略回测执行与结果展示（含至少一种可运行的默认策略或回测管线）。
- 在**量化页面**（`templates/quant.html`）将「策略回测」从占位改为**可进入的回测入口**（链接至回测页面或同页锚点/内嵌，以实现为准）。
- 新增**回测说明文档**（如 `Docs/量化回测系统.md`）：说明范围、使用方式、限制与扩展点。
- 按需新增 Flask 路由与静态/模板资源；若需持久化回测任务或结果，须**同时**满足 MySQL 与 SQLite 的 schema 约定（与项目数据库双引擎规范一致）。

## Capabilities

### New Capabilities

- `quant-backtest`: 量化策略回测的执行、结果展示及与现有行情/指标数据的集成边界；包含 Web 入口与文档层面的用户可见行为。

### Modified Capabilities

- （无）本次为新增能力，不修改既有已归档 spec 的需求行为。

## Impact

- **后端**：`routes.py`（或等价蓝图）新增页面/API；可能新增 `Core/`、`Managers/` 或独立包下的回测引擎模块。
- **前端**：`templates/quant.html` 及新增回测相关模板/静态资源。
- **数据**：回测优先复用现有 K 线与指标计算；若新增表或字段，需同步 MySQL 与 SQLite 迁移/建表逻辑。
- **文档**：`Docs/` 下新增回测说明；可选更新 `Docs/项目基础框架.md` 中的路由表。
- **依赖**：原则上复用现有 Python 栈（Flask、pandas/numpy、TA-Lib/MyTT 等）；若引入新库，在 `requirements.txt` 中声明。
