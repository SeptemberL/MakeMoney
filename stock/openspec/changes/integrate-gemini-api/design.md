## Context

项目为基于 Flask 的股票与信号系统（`main.py` 创建应用、`routes.py` 注册蓝图），配置通过 `config.ini` 与单例 `Config`（`config.py`）读取。当前 `requirements.txt` 未包含 Google AI 相关依赖。你希望接入 Gemini 作为后续多种「分析/解读」能力的统一后端，因此本阶段以**服务端集成与边界**为主，避免各业务重复实现鉴权、超时与密钥管理。

## Goals / Non-Goals

**Goals:**

- 提供**单一、可测试**的 Gemini 调用封装（同步调用即可；若后续需要再引入队列/流式）。
- 明确 **API Key 与模型标识** 的配置来源与优先级（环境变量优先于本地配置文件中的非敏感项）。
- 约定 **超时、可观测日志与错误映射**，便于路由层或定时任务一致处理。
- 与现有技术栈（Flask 3.x、Python 3）兼容的依赖选型。

**Non-Goals:**

- 不实现具体「股票分析 UI」或某一类分析 prompt 的产品化流程（留给后续变更）。
- 不要求多租户、计费仪表盘或模型微调管线。
- 不在本阶段强制引入向量库/RAG 基础设施（可作为后续 Open Question）。

## Decisions

1. **SDK 选型：优先使用 Google 官方 `google-genai`（Google Gen AI SDK for Python）**  
   - **理由**：官方维护、与 Gemini API 演进同步、减少手写 REST 与签名细节。  
   - **备选**：手写 `requests` 调 REST — 灵活但更易在鉴权/字段变更上碎裂，维护成本高。

2. **密钥来源：环境变量优先，其次可选 `config.ini` 仅作开发便利（且文档明确不推荐生产）**  
   - **理由**：与 12-factor 一致，避免密钥进入 git；与现有 `Config` 模式可并存。  
   - **建议变量名**：`GEMINI_API_KEY` 或 Google 文档中的 `GOOGLE_API_KEY`（实现时二选一并在代码中固定优先级，避免歧义）。

3. **模块位置：新建独立包或 `Managers/` 下子模块（例如 `Managers/gemini_client.py` 或 `gemini/` 包）**  
   - **理由**：与 Flask 路由解耦，便于单元测试与在调度任务中复用。  
   - **备选**：直接在 `routes.py` 调用 SDK — 快速但难以测试与复用。

4. **默认调用形态：`generate_content` 文本入参 / 文本出参**  
   - **理由**：覆盖大多数后续「分析说明」场景；结构化输出（JSON schema）可作为后续增强。  
   - **流式**：非本阶段必须；若 SDK 支持，可在封装层预留参数默认关闭。

5. **超时与重试**  
   - **超时**：可配置（如 `config.ini` 的 `[GEMINI] timeout_seconds` 或环境变量），默认合理值（例如 60s，最终以实现为准）。  
   - **重试**：对可判定为瞬时网络/5xx 的错误做有限次指数退避（例如最多 2 次），对 4xx/鉴权错误不重试。

## Risks / Trade-offs

- **[风险] 外网不可达或 DNS 被拦** → **缓解**：部署文档说明出站要求；本地开发可提供「跳过真实调用」的 mock 或集成测试打桩（后续任务可选）。
- **[风险] API Key 泄露** → **缓解**：禁止记录完整密钥；禁止把密钥下发到浏览器；代码审查检查日志脱敏。
- **[风险] 费用与配额** → **缓解**：在封装层记录每次调用的模型 id 与 token/字符粗粒度指标（若 SDK 暴露）；后续在业务层加限流/开关。
- **[权衡] 同步调用阻塞请求线程** → **缓解**：首版仅用于低频管理端或显式触发的分析；高 QPS 场景后续改为后台任务。

## Migration Plan

1. 合并代码后，在运行环境设置 API Key 与（可选）模型名。  
2. 安装更新后的 `requirements.txt` 依赖并重启服务。  
3. 回滚：移除环境变量中的密钥并回退到不含 Gemini 依赖的版本；无数据库结构变更则无需数据迁移。

## Open Questions

- 生产环境更倾向 **Vertex AI**（GCP 服务账号）还是 **Google AI Studio / API Key**？本设计默认 API Key 路径；若选 Vertex，需调整认证与客户端初始化。  
- 是否需要 **代理** 访问 Google API（部分网络环境）；若需要，是否在封装层支持 HTTP 代理环境变量。  
- 后续分析功能是否要求 **结构化 JSON 输出** 与 schema 校验；若会，封装接口需扩展返回类型。
