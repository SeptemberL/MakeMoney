## ADDED Requirements

### Requirement: 列排序三态

自选列表页（`templates/watchlist.html`）对下列表头 MUST 支持独立的排序循环，且各列排序状态互不覆盖（一次仅一列处于升序或降序，或扩展为「多列」——本需求采用**单列排序**：激活一列升/降序时，其他列自动回到无排序）：

- 股票名称（按名称主行文本）
- 当日价格
- 涨跌幅
- 分时均价（按主行均价数值；展示为 `-` 时视为缺失值）

每种列的表头交互 MUST 在三态间循环：**无排序** → **升序** → **降序** → **无排序**。表头 MUST 以可见指示区分当前态（如图标 ▲/▼ 或等价样式）。

#### Scenario: 无排序时保持列表顺序

- **WHEN** 所有列均为无排序态
- **THEN** 表格行顺序 MUST 与当前内存/持久化中的自选条目顺序一致（与添加、合并持仓、上次拖拽保存后的顺序一致）

#### Scenario: 升序排列价格

- **WHEN** 用户在「当日价格」列进入升序态
- **THEN** 行 MUST 按解析得到的现价数值从小到大排列；缺失值（无有效报价）MUST 排在同一规则下的固定一侧（实现选定后写入代码注释并在本规格中保持一致，建议置底）

#### Scenario: 切换列排序取消上一列

- **WHEN** 用户从「涨跌幅」降序切换到「分时均价」升序
- **THEN** 表格 MUST 仅按「分时均价」升序排列，涨跌幅列 MUST 回到无排序指示

### Requirement: 无排序时行号列拖拽重排

在**所有列均为无排序**时，最左侧行号单元格 MUST 作为拖拽源，允许用户将整行拖动到另一行位置并释放以插入到该位置；拖拽过程中 SHOULD 提供视觉反馈（如占位或高亮）。

在任**一列处于升序或降序**时，行号列 MUST NOT 允许拖拽重排（`draggable` 为 false 或等价行为），或拖拽不产生顺序变更。

#### Scenario: 拖拽后顺序持久化

- **WHEN** 用户在无排序态完成一次有效拖拽
- **THEN** 内存中的自选条目顺序 MUST 与 DOM 一致，且 MUST 调用与现有逻辑一致的持久化（已登录 `PUT /api/watchlist`，未登录 `localStorage`）

#### Scenario: 排序态禁止拖拽

- **WHEN** 当前存在激活的升序或降序列
- **THEN** 用户无法通过行号拖拽改变行顺序（或拖拽无效果）

### Requirement: 与行情刷新兼容

定时或手动的 `refreshQuotesOnce` MUST 在排序或拖拽后仍能更新各行情单元格，且不得在无用户操作下打乱**无排序**态下的用户自定义顺序（除非实现主动选择了「刷新时重建表」——若重建，则 MUST 在刷新后按当前 `items` 顺序重绘行并恢复排序态展示）。

#### Scenario: 无排序下刷新不改顺序

- **WHEN** 无排序态下行情定时刷新
- **THEN** 行相对顺序 MUST 不变，仅单元格数值更新
