## Why

目前涨停复盘结果以 JSON 为主，阅读与分享成本高；将结果渲染为统一风格的 PNG 图，可以直接在聊天/飞书/微信群中转发，提升复盘效率与传播一致性。

## What Changes

- 新增“将指定交易日的涨停复盘结果渲染为 PNG 图片”的能力。
- 支持从已有的 `daily_limitup_report` 输出（JSON）读取数据，生成单张汇总图（或按配置生成多张分页图）。
- 提供任务/命令入口，便于每日 17:00 后自动生成并可选推送到通知渠道。

## Capabilities

### New Capabilities
- `daily-limitup-report-png`: 将涨停复盘结构化数据渲染为可分享的 PNG 图片（含标题、日期、表格/卡片、关键字段）。

### Modified Capabilities
- （无）

## Impact

- 增加图片渲染依赖（例如 Pillow / matplotlib / playwrght 之一）及其运行环境要求（字体、中文渲染）。
- 增加一个新的定时任务（或挂接到现有 17:05 的涨停复盘任务后置步骤）。
- 若需要推送图片到现有通知通道（飞书/微信），需要复用或扩展现有通知模块的“发送图片”能力。
