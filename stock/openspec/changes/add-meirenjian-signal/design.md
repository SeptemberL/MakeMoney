## Context

现有项目具备股票筛选/信号的基础能力，但对“形态类信号”缺少统一、可复用且口径明确的实现与参数规范。“美人肩”属于典型的价格结构形态，识别过程涉及：

- 时间窗口与多段结构（左肩/头部/右肩或等价分段）
- 关键点选择（局部高低点、颈线/支撑线）
- 容错与阈值（高度差、回撤幅度、成交量约束、形态持续时间）

本变更在不破坏现有筛选框架的前提下，引入一个新的形态信号能力 `signal-meirenjian`，并把形态识别的输入输出与参数统一到 specs 中。

## Goals / Non-Goals

**Goals:**
- 将“美人肩”形态识别抽象为独立信号能力（可配置参数 + 明确输入输出），可用于扫描/选股/回测。
- 与现有信号体系一致的调用方式（同一注册/路由/参数注入/结果结构），便于组合多个信号。
- 产出对调参友好的中间量（关键拐点、颈线、形态评分/置信度等），便于回测分析与可视化（是否记录由实现侧决定，但 spec 需定义可选输出）。

**Non-Goals:**
- 不在本次变更内实现完整的 UI 可视化或 K 线形态标注工具（如有需要，另立变更）。
- 不引入新的外部数据源（以现有 OHLCV 数据为基础；复权/周期口径在 spec 中明确）。
- 不在设计阶段绑定到某一种唯一算法实现（允许后续在不改变 spec 的前提下优化识别算法）。

## Decisions

- **信号能力边界**：把“美人肩”作为一个独立 capability（`signal-meirenjian`），其要求由 `specs/signal-meirenjian/spec.md` 定义；实现应只依赖通用行情数据与统一的信号框架，不直接耦合某个策略。
  - **Alternative**：把形态逻辑嵌入某个策略或筛选脚本中。缺点是复用差、口径难统一、测试困难。

- **输入数据口径**：默认以日线 OHLCV 为输入，支持指定时间窗口（例如最近 N 根 K 线），在 spec 中明确：
  - 是否必须复权（前复权/后复权/不复权）
  - 停牌/一字板/极端数据的处理
  - 最小数据长度与不足时的返回约定
  - **Alternative**：多周期同时识别（周线+日线）。复杂度高，建议后续在不破坏接口的前提下扩展。

- **输出结构**：信号输出至少包含：`hit`（是否命中）、`score`（可选，0~1 或 0~100）、以及关键点坐标（可选输出字段，但在 spec 中要标准化字段名与含义）。
  - **Alternative**：只返回命中/不命中。缺点是不可解释、调参成本高、难以回测诊断。

- **参数化与容错**：将所有阈值与窗口均参数化（并提供默认值），例如肩高差允许比例、头部突出幅度、颈线斜率容忍、最小形态跨度等。
  - **Alternative**：固定阈值。缺点是适配性差，跨市场/跨品种泛化差。

## Risks / Trade-offs

- **[形态定义歧义]** “美人肩”在不同交易者口径下定义差异较大 → **Mitigation**：在 spec 中给出清晰的数学化判定与示例约束，并将弹性部分做成参数；必要时支持多种 mode（如 `strict`/`loose`）。

- **[过拟合/误报]** 形态识别容易在噪声数据上产生大量误报 → **Mitigation**：引入评分机制与最小置信度阈值；可选成交量/波动率过滤；回测阶段统计命中后的分布以校准默认参数。

- **[关键点提取稳定性]** 局部极值识别对参数敏感 → **Mitigation**：采用稳定的 pivot 规则（如 fractal/pivot-high-low）并固定优先级；同时输出关键点以便诊断。

- **[性能]** 批量扫描全市场时，形态识别可能成为瓶颈 → **Mitigation**：窗口化计算、预过滤（如最近 N 日波动/趋势）、必要时缓存中间量。
