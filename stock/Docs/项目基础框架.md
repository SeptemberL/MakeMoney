## 六、核心模块说明

### 6.1 stock_list_manager.py — 股票列表与数据管理

项目最核心的模块，负责：
- `get_all_a_stocks()` — 从 akshare 拉取全市场 A 股列表，筛选主板(00/60)、创业板(300)、科创板(688)
- `update_stock_list()` — 遍历股票列表，逐只更新历史数据 + 检查信号
- `update_stock_history()` — 单只股票增量更新（检查表是否存在 → 判断最新日期 → 拉取差异数据 → 写入 DB）
- `get_stock_data()` — 从 DB 读取指定天数的行情数据

每只股票的数据存储在独立表 `stock_{code}_{SH|SZ}` 中，表名通过 `_get_stock_table_name()` 生成。

### 6.2 stock_analyzer.py — 技术指标计算

使用 TA-Lib 计算以下指标：
- **均线**: MA5/10/20/30/60/99/250
- **BOLL**: 通过 Signal_BOLL3 计算（EMA中轨 + 2/3倍标准差轨道）
- **MACD**: DIF/DEA/柱状值，通过 Signal_DLJG2 计算底背离
- **KDJ**: K/D/J 值（9,3,3 参数）
- **RSI**: 6/12/24 日 RSI
- **成交量**: 5/10/20 日成交量均线 + OBV
- **其他**: ATR、CCI、DMI(PDI/MDI/ADX)、趋势强度

### 6.3 信号策略

**Signal_BOLL3** — 布林带 3σ 策略
- 中轨 = EMA(close, 30)
- 标准差 = STD(30) × (1 + ATR修正因子)
- 四种信号（均需量比配合）：
  - `UPPER_BREAK`: 收盘价在 2σ~3σ 之间 且 量比>1.2
  - `UPPER_REVERSE`: 收盘价突破 3σ 上轨 且 量比>1.5
  - `LOWER_BREAK`: 收盘价在 -2σ~-3σ 之间 且 量比>1.2
  - `LOWER_REVERSE`: 收盘价跌破 -3σ 下轨 且 量比>1.5

**Signal_DLJG2** — MACD 底背离钝化预警
- 追踪死叉间隔（N1/N2/N3），计算相邻死叉间的价格低点与 DIF 低点
- 检测直接底背离（价格创新低，DIF 未创新低）和隔峰底背离
- 输出三种状态：底钝化开始 → 底钝化形成 → 底钝化消失

### 6.4 NGA 爬虫子系统

独立的论坛帖子监控系统：
- **nga_monitor.py**: 后台线程按 `poll_interval` 秒轮询所有 auto_run=1 的帖子
- **nga_crawler.py**: 单帖增量爬取（从 DB 记录的进度页码开始），解析 NGA JSON 响应
- **nga_db.py**: 管理楼层存储、发送日志、帖子配置
- 符合 `watch_author_ids` 的楼层内容通过 wxauto 推送到配置的微信群

### 6.5 定时调度系统

基于 APScheduler，通过 `configs/tasks_config.yaml` 配置：
- 支持 cron / interval / date 三种触发模式
- 通过 `module_path` + `function_name` 字符串动态加载执行函数
- TaskManager 管理任务的添加/暂停/恢复/移除

## 七、配置文件说明

### config.ini

```ini
[DEFAULT]
UPDATE_STOCKS_DELAY = 0.2        # 批量更新时每只股票的请求间隔（秒）

[DATABASE]
DB_TYPE = mysql                   # mysql / sqlite
DB_HOST / DB_PORT / DB_NAME ...   # MySQL 连接参数
DB_PATH = database/stock.db      # SQLite 文件路径

[WX]
EnableWX = 0                      # 是否启用微信推送（0=关闭）
TestImageUrl = ...                # 测试图片 URL
TestImageSendTo = 光影相生         # 测试发送目标

[TUSHARE]
TOKEN = ...                       # Tushare Pro 数据接口 Token
```

### configs/tasks_config.yaml

定时任务配置，每个任务包含：task_id、task_name、module_path、function_name、trigger_type、trigger_args、enabled

### configs/nga.yaml

NGA 爬虫配置，包含 auth（Cookie 认证）和 settings（轮询间隔、User-Agent 等）

### configs/send_message_group.yaml

微信群组映射，groups 列表中每项包含 group_id 和 chat_list（微信聊天名称列表）
