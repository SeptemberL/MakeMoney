# 自选列表

## 存储说明

| 场景 | 存储位置 | 跨设备 |
|------|----------|--------|
| **未登录** | 浏览器 `localStorage`（`watchlist_items_v1`） | 否 |
| **已登录** | 数据库表 `user_watchlist`（按 `user_id`） | 是（同一账号、同一套后端库） |

## API

- `GET /api/watchlist`：读取当前登录用户的自选（需 session，未登录返回 401）。
- `PUT /api/watchlist`：整体替换自选列表，body：`{ "items": [ { "stock_code", "stock_name" }, ... ] }`。

## 行为说明

- 打开自选页时：若已登录，先从数据库加载，并与本机 `localStorage` 合并（本机多出的代码会一并写入库，便于从「仅本机」迁移到云端）。
- 仍会合并**当前登录用户**的持仓股票（只加不删，接口 `GET /api/positions/simple`）；未登录时该接口返回空列表，故不会合并持仓。合并结果写回数据库或 `localStorage`（取决于是否登录）。

## 数据库

- 表名：`user_watchlist`（MySQL / SQLite 均已建表，见 `database/database.py` 中 `init_database`）。
