# 量化回测系统

## 功能概述

本系统在本仓库内提供**基于本地日线的策略回测**能力，当前内置 **双均线（金叉买入 / 死叉卖出）** 示例策略：在指定股票代码与日期区间内，按收盘价模拟成交，并输出累计收益率、最大回撤、期末权益与交易明细。

回测**不依赖**额外商业回测云服务，数据来自项目已落库的 `stock_<代码>_<SH|SZ>` 表（与「股票列表 + 历史更新」流程一致）。

## 使用步骤

1. 确保目标标的已在系统中拉取过日线（与首页/单股分析使用同一套本地表）。
2. 浏览器打开 **量化** 页：`/quant`，在「策略回测」卡片点击 **进入回测**；或直接访问 `/quant/backtest`。
3. 填写 **6 位股票代码**、**开始/结束日期**（`YYYY-MM-DD`）、短期/长期均线周期（默认 5 / 20）、初始资金。
4. 点击 **运行回测**，在右侧查看摘要与交易明细 JSON 行。

API（可选）：`POST /api/quant/backtest`，JSON 体字段示例：

```json
{
  "stock_code": "600000",
  "start_date": "2024-01-01",
  "end_date": "2025-01-01",
  "short_window": 5,
  "long_window": 20,
  "initial_cash": 100000
}
```

成功时返回 `success: true` 与 `data`（含 `total_return_pct`、`max_drawdown_pct`、`trades`、`equity_curve` 等）；失败时 `success: false` 与 `message`。

## 数据与假设

- **数据粒度**：日线；若区间内本地无数据，将提示无法回测。
- **成交价格**：当日收盘价；**未**模拟滑点、停牌、涨跌停不可成交等细节。
- **佣金**：按单边费率近似（默认万分之五），买入与卖出均扣减；实际费率以券商为准。
- **复权**：与本地表存储一致（项目拉取流程使用 akshare 前复权时，回测即为前复权序列）。

## 限制与扩展

- **当前不落库**：回测任务与结果不写入数据库；刷新页面即丢失，仅当次会话展示。
- **策略**：当前仅为双均线示例；扩展时可向 `backtest/` 模块增加策略并实现路由侧策略选择参数。
- **性能**：长区间、全市场批量回测可能占用较多 CPU；建议先缩小日期范围验证。
