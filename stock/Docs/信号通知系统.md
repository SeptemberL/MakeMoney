股票信号通知系统

这个系统通过对股票设定一些信号，在交易时间每次刷新股票判断一次股价是否触发信号了，如果触发信号了，则向对应的群组发送 message

1、创建这个系统的基础框架
    1.1 留出接口，用于在其他界面使用，点击界面按钮后，打开一个悬浮页面，可以填写信号逻辑
    1.2 创建信号系统的基础框架， 这个信号系统保存了当前检查信号的股票， 发送群组， 信号类型， 每种信号类型的参数， 发送消息的模板，模板可以通过一些富文本来替换股票代码、股票价格， 发送类型（触发就发送、是否每隔一段时间发送，子类型可以覆盖这个发送类型，例如到上下限的，到上限了发送一次，下限发送一次这种。
    1.3 先创建一个最基础的信号类型，股价到一个价位范围内了，就对群组发信号，到上限了发送一次，到下限了发送一次， 股票价格低于下限，则股价高于上下限的时候触发，股票价格低于下限，则高于上下限的时候触发一次。
    
信号基类: SingleBase
    接口: Init 初始化
         Update 每隔一段时间更新价格，检查是否触发信号
         Trigger 触发信号
         SendMessage 向对应群组发送信号
信号类型: SingleType

信号管理器: SingleManager
    

2、消息模板富文本变量说明

消息模板支持使用花括号变量（如 `{stock_code}`）动态替换信号上下文信息。

2.1 可用变量

- `{stock_code}`: 股票代码，例如 `600000`
- `{stock_name}`: 股票名称，例如 `浦发银行`
- `{signal_type}`: 信号类型，例如 `price_range`
- `{price}`: 当前触发价格（可配格式，例如 `{price:.3f}`）
- `{boundary}`: 触发边界，`lower` 或 `upper`
- `{lower}`: 区间下限（可配格式，例如 `{lower:.3f}`）
- `{upper}`: 区间上限（可配格式，例如 `{upper:.3f}`）
- `{time}`: 触发时间字符串，例如 `2026-03-20 10:30:00`

到价提醒类型 `price_level_interval` 额外变量：

- `{target_price}`: 规则中配置的目标价
- `{mode}`: 触发模式字符串，`at_or_above` 或 `at_or_below`
- `{mode_label}`: 可读说明，如「高于等于目标价」「低于等于目标价」
- `{lower}` / `{upper}`: 与模板兼容，均填为目标价

黄金分割类型 `fibonacci_retrace` 额外常用变量：

- `{zone_label}`: 中文判定标签，如 `常规买点`、`强支撑`、`黄金坑`
- `{fib_low}` / `{fib_high}`: 参数中的最低/最高价
- `{level_382}` / `{level_500}` / `{level_618}`: 计算得到的三档价格

三类条件可配置**三条独立消息模板**，保存在 `params.templates` 中，键名为：

- `normal_buy`：常规买点（当前价低于 0.382 档且高于 0.5 档区间触发）
- `strong_support`：强支撑（低于 0.5 档且高于 0.618 档区间触发）
- `golden_pit`：黄金坑（低于 0.618 档触发）

若某一键未填写，则回退使用规则主字段 `message_template`。

2.2 推荐模板示例

- 简洁版  
  `股票 {stock_code}({stock_name}) 触发{signal_type}，现价 {price:.3f}`

- 区间提醒版  
  `股票 {stock_code}({stock_name}) 触发{signal_type}信号，当前价格: {price:.3f}，触发边界: {boundary}，区间[{lower:.3f}, {upper:.3f}]，时间: {time}`

- 可读性增强版（多行）  
  `【信号提醒】`  
  `股票: {stock_code} {stock_name}`  
  `类型: {signal_type}`  
  `价格: {price:.3f}`  
  `边界: {boundary}`  
  `区间: [{lower:.3f}, {upper:.3f}]`  
  `时间: {time}`

2.3 使用约定

- 模板变量名必须与系统字段一致，区分大小写。
- 不存在的变量会导致渲染失败或为空，新增变量时需同步更新代码与文档。
- 建议价格类字段统一保留 2~3 位小数，时间统一使用 `YYYY-MM-DD HH:MM:SS`。

3、通知频率约定（当前实现）

**价格区间 `price_range`**

- 价格命中区间 `[lower, upper]` 时触发通知。
- 同一条规则按自然日去重：每天最多通知一次。
- 触发后会写入 `signal_rule_state`，字段 `last_notified_date` 记录最后通知日期。
- 次日再次命中区间时可继续通知，不会因为历史已通知而永久静默。

**到价提醒 `price_level_interval`**

- 参数：`params.target_price`、`params.mode`（`at_or_above`：现价≥目标价；`at_or_below`：现价≤目标价）。
- 必须使用 `send_type=interval`，且 `send_interval_seconds` 为**不小于 60** 的整数；条件持续成立时，每隔该秒数至多发送一条（依赖行情刷新频率，为「至少间隔」）。
- 运行态写入 `signal_rule_state.state_json`：`last_sent_at`（上次成功发送时间）、`last_price`。
- 条件由真变假时不发送；再次变真时仍须满足与上次发送之间的完整间隔（不通过闪断绕过节流）。


4、 当前信号类型
  1） 根据一个价格区间对价格监控进行触发
      输入参数： 价格上限、价格下限
      是否完成：已完成
      信号类型：price_range
  
  2） 根据黄金分割率来计算信号
      输入参数： 价格最低点、价格最高点
      计算公式： 价格最高 - （价格最高点-价格最低点）* x   x为：0.382， 0.5， 0.618  计算出几个价格
      如果当前价格 小于 0.382的价格，高于0.5 标记为 常规买点， 如果当前价格小于0.5 大于0.618，为强支撑， 低于0.618附近为黄金坑
      每当价格低于其中一个档位时则发起message通知。
      在填写好上下限后，在下一行自动计算出三个档位的价格显示出来

  3） 到价提醒（到价后按秒重复提醒）
      信号类型：`price_level_interval`（内部标识不变）
      输入参数：目标价 `target_price`、触发条件 `mode`（`at_or_above` / `at_or_below`）
      发送：`send_type` 固定为 `interval`，`send_interval_seconds` ≥ 60
      是否完成：已完成

4.2 信号填写页按类型动态控件（已实现）

- 新增 `signal_type` 选择：`price_range`、`fibonacci_retrace`、`price_level_interval`（界面文案：**到价提醒**）。
- 选择 `price_range` 时显示参数：`lower`、`upper`。
- 选择到价提醒（`price_level_interval`）时显示：目标价、`mode`，并强制间隔发送与「发送间隔（秒）」输入；同时显示与 `price_range` 相同的**消息模板**文本框，可编辑发送内容，保存至规则字段 `message_template`（创建/更新 API 均支持）。常用占位符见上文「到价提醒类型」变量说明（如 `{target_price}`、`{mode_label}`、`{price}`、`{time}`）。
- 选择 `fibonacci_retrace` 时显示参数：`low`、`high`，并自动计算/展示 `0.382`、`0.5`、`0.618` 三档位价格。
- 黄金分割支持 `params.templates.normal_buy` / `strong_support` / `golden_pit` 三条消息模板。
- 前端提交时根据类型构造不同 `params`，后端按 `signal_type` 做参数校验与信号实例化。

4.3 自选列表中的多信号管理（已实现）

- 同一 `stock_code` 可有多条 `signal_rule`。
- 自选列表打开信号弹窗后，顶部列表展示该股票全部信号（含停用）；`GET /api/signal_notify/signals?stock_code=...&only_active=false`。
- 点击一行加载下方表单并「保存修改」；点「新建信号」清空规则 ID 后「保存（新建）」新增一条。
- 编辑已有规则时，前端锁定「信号类型」不可改（与当前 PUT 接口一致）；新建时可自由选择类型。

5、推送通道（config.ini）

- `[NOTIFY]`：`channel = wechat`（默认）或 `feishu`。
- `wechat`：沿用本机微信 + `wxauto` 与 `WXGroupManager` 按 `group_id` 解析群聊名发送；无微信实例时仅打日志（模拟）。
- `feishu`：在 `[FEISHU]` 中配置 `webhook_url`（飞书机器人完整 Webhook），通过 HTTPS POST 发送 JSON：`{"msg_type":"text","content":{"text":"渲染后的正文"}}`。若机器人启用了**签名校验**，在 `sign` 中填写控制台给出的密钥，请求体会额外包含当前秒级 `timestamp` 与按飞书规则计算的 `sign` 字段。可选 `timeout_seconds`（默认 10）。
- **V1**：飞书为单 Webhook 时，同一条信号正文在一次 `send_message` 内无论多少个 `group_id` 仅 POST 一次；请勿将 Webhook 提交到版本库。
- **统一入口**：文本类通知（信号、NGA 楼层推送、`tasks/check_lof`、批量更新汇总等）均经 `Managers/notify_channel.py` 的 `send_notify_to_group` / `send_notify_fallback`，与上述通道一致。
- 无 `group_id` 的汇总消息（如 `/api/update_all_stocks`）：微信通道可配置 `fallback_group_id` 指向已有 `message_group`；未配置则发到 `[WX] TestImageSendTo` 指定会话。
